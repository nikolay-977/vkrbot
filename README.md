# VKRBOT — Диалоговая система для взаимодействия студентов и научных руководителей

[![Java Version](https://img.shields.io/badge/Java-17-blue.svg)](https://adoptium.net/)
[![Spring Boot](https://img.shields.io/badge/Spring%20Boot-3.1.5-brightgreen.svg)](https://spring.io/projects/spring-boot)
[![Telegram Bot API](https://img.shields.io/badge/Telegram%20Bot%20API-6.9.7.1-blue)](https://core.telegram.org/bots/api)

**VKRBOT** — это Telegram-бот, созданный для формализации и упорядочивания коммуникации между студентом и научным руководителем при подготовке выпускных квалификационных работ (ВКР), курсовых проектов и научных исследований.

Система позволяет:
- создавать дедлайны и задачи с чёткими критериями выполнения;
- отслеживать статусы задач (создана, в работе, на проверке, доработка, принята, отменена);
- обмениваться комментариями и файлами, привязанными к конкретным задачам;
- получать автоматические уведомления в Telegram и на email;
- управлять пользователями через веб-админку (Spring MVC + Thymeleaf).

## Архитектура

Система состоит из трёх основных компонентов:

1. **Telegram-бот** (`TelegramBotService`) — обрабатывает команды пользователей, управляет состояниями диалога, отправляет уведомления.
2. **Веб-админка** (Spring MVC + Thymeleaf) — позволяет администратору управлять пользователями, назначать руководителей, генерировать токены.
3. **База данных** (PostgreSQL) — хранит пользователей, чаты, дедлайны, задачи, комментарии.
4. **Redis** — централизованное хранилище состояний многошаговых сценариев (конечных автоматов), обеспечивает отказоустойчивость и горизонтальное масштабирование.

Все компоненты контейнеризированы с помощью Docker и могут быть развёрнуты одной командой.

## Установка и запуск

### Локальная разработка (dev-профиль)

Для разработки используется `docker-compose.dev.yml`, который поднимает только PostgreSQL и Redis, а Spring Boot приложение запускается локально через IDE или Maven.

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/nikolay-977/vkrbot.git
   cd vkrbot
   ```

2. **Запустите контейнеры для разработки:**
   ```bash
   docker-compose -f docker-compose.dev.yml up -d
   ```
    - PostgreSQL доступен на порту `5433` (логин: `admin_user_dev`, пароль: `admin_password_dev`, БД: `admin_db_dev`)
    - Redis доступен на порту `6379`

3. **Настройте переменные окружения** (создайте файл `.env` в корне проекта или экспортируйте):
   ```bash
   export EMAIL_USER=your_email@yandex.ru
   export EMAIL_PASSWORD=your_app_password
   export TELEGRAM_BOT_TOKEN=your_bot_token
   export TELEGRAM_BOT_USERNAME=your_bot_username
   ```

4. **Запустите Spring Boot приложение:**
   ```bash
   mvn spring-boot:run -Dspring-boot.run.profiles=dev
   ```
   Или запустите `VkrBotApplication.java` в вашей IDE с профилем `dev`.

5. **Веб-админка** станет доступна по адресу: [http://localhost:8080/login](http://localhost:8080/login)  
   Логин: `admin@admin.com`, пароль: `admin123` (создаётся автоматически при первом запуске).

### Продакшен-режим (Docker)

Для полноценного развёртывания всех компонентов (включая Spring Boot приложение) используется `docker-compose.yaml`.

1. **Скопируйте файл `.env.example` в `.env` и заполните реальные значения:**
   ```bash
   cp .env.example .env
   nano .env
   ```
   Минимально необходимые переменные:
   ```ini
   DB_NAME=admin_db
   DB_USER=admin_user
   DB_PASSWORD=admin_password
   EMAIL_USER=your_email@yandex.ru
   EMAIL_PASSWORD=your_app_password
   TELEGRAM_BOT_TOKEN=your_bot_token
   TELEGRAM_BOT_USERNAME=your_bot_username
   ```

2. **Соберите и запустите все сервисы:**
   ```bash
   docker-compose up -d --build
   ```

3. **Проверьте состояние контейнеров:**
   ```bash
   docker-compose ps
   ```

4. **Веб-админка** будет доступна по адресу: [http://localhost:8080](http://localhost:8080) (или порт, указанный в конфигурации).

> **Примечание:** Для продакшена рекомендуется изменить пароль администратора и БД на более сложные.

### Telegram бот

1. Напишите [@BotFather](https://t.me/botfather) в Telegram.
2. Создайте нового бота командой `/newbot`.
3. Скопируйте полученный **токен** и **username**.
4. Установите значения в переменные окружения `TELEGRAM_BOT_TOKEN` и `TELEGRAM_BOT_USERNAME`.

### Email уведомления

В качестве SMTP-провайдера используется **Яндекс.Почта** (настройки уже в `application.yml`). Для работы:

1. Включите доступ к почте по протоколу SMTP в настройках вашего аккаунта Яндекса.
2. Создайте **пароль приложения** в разделе «Безопасность» → «Пароли приложений».
3. Укажите `EMAIL_USER` = ваш email, `EMAIL_PASSWORD` = пароль приложения.

## Резервное копирование и восстановление

В проекте предусмотрены shell-скрипты для бэкапа и восстановления базы данных PostgreSQL.

### Бэкап

```bash
./backup.sh
```
- Создаёт дамп базы данных `admin_db` (пользователь `admin_user`) в папку `./backups/`.
- Имя файла формата `backup_YYYYMMDD_HHMMSS.sql.gz`.
- Автоматически удаляет бэкапы старше 7 дней.

### Восстановление

```bash
./restore.sh ./backups/backup_20260512_120000.sql.gz
```
- Восстанавливает базу из указанного файла (может быть как `.sql`, так и `.sql.gz`).

> **Требования:** Скрипты предполагают, что контейнер PostgreSQL называется `admin-app-postgres`. Если имя отличается, измените его в скриптах.

## Развертывание обновлений

Скрипт `deploy.sh` автоматизирует процесс обновления приложения:

```bash
./deploy.sh
```

Что делает скрипт:
1. Собирает приложение через `mvn clean package -DskipTests`.
2. Останавливает старые контейнеры (используя `docker-compose.dev.yml`).
3. Запускает новые контейнеры с пересборкой (`--build`).
4. Показывает статус контейнеров.

Для продакшен‑среды измените в скрипте файл на `docker-compose.yaml` или создайте отдельный скрипт.

## Использование

### Регистрация пользователя

1. Администратор заходит в веб-админку (`/admin/users/create`) и создаёт пользователя (студента или руководителя), указывая email.
2. Система генерирует уникальный токен и отправляет его на email пользователя.
3. Пользователь открывает Telegram, находит бота (по username), отправляет команду `/start`.
4. В ответ бот просит ввести токен. После успешной авторизации бот показывает меню, соответствующее роли.

### Основные функции

| Роль            | Доступные действия                                                                 |
|-----------------|------------------------------------------------------------------------------------|
| Студент         | Просмотр диплома, дедлайнов и задач, выполнение задач, отправка на проверку, комментарии, файлы |
| Руководитель    | Просмотр своих студентов, создание дедлайнов и задач, проверка задач по критериям, комментарии, файлы |
| Администратор   | Управление пользователями через веб-админку (создание, редактирование, блокировка, назначение руководителей, генерация токенов) |

### Пример рабочего процесса

1. Руководитель создаёт дедлайн «Сдача первой главы», устанавливает дату.
2. Внутри дедлайна создаёт задачу «Написать обзор», добавляет критерии: «не менее 5 источников», «ссылки по ГОСТ».
3. Студент видит задачу, берёт в работу, выполняет, прикрепляет файл и отправляет на проверку.
4. Руководитель получает уведомление, проверяет, отмечает критерии (по одному или все).
5. Руководитель принимает задачу (статус `DONE`) или отправляет на доработку с комментарием.
